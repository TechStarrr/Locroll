## 🎯 Project Overview

Build **Remlo** — a global payroll platform where AI agents autonomously execute, verify, and settle employee salaries using Locus payment infrastructure. This is not a mock-up. Every payment must flow through real Locus APIs with real USDC on Base.

The core thesis: **payroll is the highest-frequency, highest-trust financial operation in any business** — and it is still done manually. Remlo makes it fully autonomous. An AI agent checks compliance, evaluates treasury readiness, batches salaries, and executes payment — all without a human clicking "confirm."

## 💳 Locus Payment Architecture

Use the **full Locus suite** as described at `docs.paywithlocus.com`. Each feature maps to a specific Remlo workflow:

### 1. Locus Wallets — Treasury & Employee Wallets
- **What**: Non-custodial smart wallets on Base with sponsored gas and subwallet support
- **How to use in Remlo**:
  - On company signup → provision a **company treasury wallet** via Locus
  - On employee invite → provision a **per-employee subwallet** via Locus
  - Fund treasury wallet with USDC; employees never manage crypto directly
- **API**: `POST /v1/wallets` to create, `GET /v1/wallets/:id/balance` for balance checks
- **Locus API key**: stored server-side in `LOCUS_API_KEY` env var (prefix: `locus_`)

### 2. Locus Payments (Send) — Payroll Execution
- **What**: Send USDC to wallet addresses or email recipients
- **How to use in Remlo**:
  - Payroll agent calls `POST /v1/payments` for each employee in the batch
  - Supports send-to-email (employees without wallets get an email claim link)
  - Every payment includes a `memo` field — encode: `{ employeeId, payrollRunId, period, currency: "USDC" }`
- **Batching**: Loop payments with idempotency keys (`payrollRunId:employeeId`) to prevent double-pays
- **Spend Policy**: bind the payroll agent to a Locus policy group with:
  - Max per-transaction: employee's monthly salary cap
  - Daily limit: total payroll budget
  - Vendor allowlist: only employee wallet addresses
  - Required justification: auto-generated by the AI agent per payment

### 3. Locus Spend Policies — Agent Guardrails
- **What**: Define spending limits, per-transaction budgets, approval thresholds, and vendor allowlists per agent identity
- **How to use in Remlo**:
  - Create a `PayrollAgent` identity in Locus dashboard
  - Bind it to a policy: `{ maxPerTx: salaryMax, dailyLimit: payrollBudget, requireJustification: true }`
  - The AI agent runs inside these guardrails — it cannot overspend, cannot pay unknown wallets
  - Every transaction produces an auditable log: who got paid, when, and why (the AI's justification)
- **This is your hackathon differentiator**: show judges a policy-bound agent that cannot go rogue

### 4. Locus Checkout — Employee Off-Ramp / Card Spend
- **What**: Stripe-style checkout SDK — accept USDC payments and route funds to Locus wallets
- **How to use in Remlo**:
  - Employees who want to spend salary immediately can use `Checkout with Locus` to top up a **Locus Virtual Debit Card**
  - Integrate Locus Checkout SDK on the employee dashboard: `import { LocusCheckout } from '@locus/checkout'`
  - Route funds from employee subwallet → virtual card for immediate USD spend
- **API**: Follow `docs.paywithlocus.com` Checkout SDK reference

### 5. Locus Virtual Cards — Employee Visa Spend
- **What**: Order prepaid virtual debit cards using USDC for online purchases
- **How to use in Remlo**:
  - Each employee can request a virtual Visa card funded from their Locus subwallet
  - Call `POST /v1/cards` with the employee's wallet ID
  - Display card details (masked PAN, CVV, expiry) in the employee dashboard
  - Employees spend in USD anywhere Visa is accepted — no crypto knowledge needed

### 6. Locus Pay-Per-Use APIs — AI Compliance Screening
- **What**: 30+ pay-per-use third-party APIs accessible from a single Locus wallet (AI models, data providers, etc.)
- **How to use in Remlo**:
  - The payroll AI agent uses Locus's pay-per-use API catalog to call **OFAC/sanctions screening APIs** per employee before executing payment
  - No separate API accounts — the agent's Locus wallet pays fractions of a cent per compliance check
  - If screening fails → block payment, flag for human review, log reason

### 7. Locus MCP — Agent Tool Interface
- **What**: MCP gateway at `https://mcp.paywithlocus.com/mcp` — lets AI agents call Locus tools natively
- **How to use in Remlo**:
  - The Claude payroll agent uses Locus MCP tools directly: `send_payment`, `check_balance`, `get_policy_status`
  - Agent is authenticated via Bearer token (`Authorization: Bearer locus_...`)
  - This enables fully autonomous payroll: Claude decides → calls MCP → Locus executes → on-chain settlement

## 🤖 AI Agent Architecture

The centerpiece of Remlo is the **PayrollAgent** — a Claude-powered autonomous agent that runs the entire payroll lifecycle.

### Agent Workflow (Tool-Use Loop)

```
PayrollAgent.run(payrollRunId) →
  1. check_treasury_balance()       [Locus wallet API]
  2. get_employee_list()            [Postgres]
  3. screen_compliance(employees)   [Locus pay-per-use API → sanctions check]
  4. validate_payroll_amounts()     [internal logic]
  5. execute_batch_payments()       [Locus send API × N employees]
  6. generate_payslips()            [Postgres + PDF]
  7. notify_employees()             [email/SMS]
  8. write_audit_log()              [Postgres]
```

## 🔌 Locus Client Implementation

```typescript
// src/lib/locus/client.ts
const LOCUS_BASE = process.env.LOCUS_API_BASE!;
const LOCUS_KEY  = process.env.LOCUS_API_KEY!;

async function locusRequest<T>(
  path: string,
  method: "GET" | "POST" | "PATCH" = "GET",
  body?: unknown
): Promise<T> {
  const res = await fetch(`${LOCUS_BASE}${path}`, {
    method,
    headers: {
      "Authorization": `Bearer ${LOCUS_KEY}`,
      "Content-Type": "application/json",
    },
    body: body ? JSON.stringify(body) : undefined,
  });
  if (!res.ok) {
    const err = await res.json();
    throw new Error(`Locus API error ${res.status}: ${JSON.stringify(err)}`);
  }
  return res.json();
}

// Wallet Operations
export const createWallet = () =>
  locusRequest<{ id: string; address: string }>("/v1/wallets", "POST");

export const getBalance = (walletId: string) =>
  locusRequest<{ usdc: number }>(`/v1/wallets/${walletId}/balance`);

// Send Payment
export const sendPayment = (params: {
  fromWalletId: string;
  toAddress?: string;
  toEmail?: string;
  amountUsdc: number;
  memo: string;
  idempotencyKey: string;
  justification: string;
}) => locusRequest<{ id: string; status: string }>("/v1/payments", "POST", params);

// Virtual Card
export const createCard = (walletId: string) =>
  locusRequest<{ id: string; last4: string; expiry: string }>("/v1/cards", "POST", { walletId });
```

## 🤖 PayrollAgent Implementation

```typescript
// src/lib/agent/payroll-agent.ts
import Anthropic from "@anthropic-ai/sdk";
import { getBalance, sendPayment } from "../locus/client";
import { prisma } from "../db/prisma";
import { PAYROLL_TOOLS } from "./tools";
import { PAYROLL_SYSTEM_PROMPT } from "./prompts";

export async function runPayrollAgent(payrollRunId: string) {
  const client = new Anthropic();
  const run = await prisma.payrollRun.findUniqueOrThrow({
    where: { id: payrollRunId },
    include: { company: true, payments: { include: { employee: true } } },
  });

  const employees = await prisma.employee.findMany({
    where: { companyId: run.companyId },
  });

  const messages: Anthropic.MessageParam[] = [
    {
      role: "user",
      content: `Execute payroll run ${payrollRunId} for ${run.company.name}. 
      Treasury wallet: ${run.company.locusWalletId}. 
      Employees to pay: ${JSON.stringify(employees.map(e => ({
        id: e.id, name: e.name, wallet: e.locusWalletId, salary: e.salaryUsdc
      })))}
      Period: ${new Date().toISOString().slice(0, 7)}`
    }
  ];

  // Agentic loop
  while (true) {
    const response = await client.messages.create({
      model: "claude-opus-4-5",
      max_tokens: 4096,
      system: PAYROLL_SYSTEM_PROMPT,
      tools: PAYROLL_TOOLS,
      messages,
    });

    // Stream agent activity to frontend via SSE (write to Redis pubsub or DB)
    await logAgentStep(payrollRunId, response);

    if (response.stop_reason === "end_turn") break;

    if (response.stop_reason === "tool_use") {
      const toolResults: Anthropic.ToolResultBlockParam[] = [];

      for (const block of response.content) {
        if (block.type !== "tool_use") continue;

        const result = await executeAgentTool(block.name, block.input as Record<string, unknown>, run);
        toolResults.push({ type: "tool_result", tool_use_id: block.id, content: JSON.stringify(result) });
      }

      messages.push({ role: "assistant", content: response.content });
      messages.push({ role: "user", content: toolResults });
    }
  }

  await prisma.payrollRun.update({
    where: { id: payrollRunId },
    data: { status: "COMPLETED", completedAt: new Date() },
  });
}

async function executeAgentTool(
  name: string,
  input: Record<string, unknown>,
  run: { company: { locusWalletId: string } }
) {
  switch (name) {
    case "check_treasury_balance":
      return getBalance(run.company.locusWalletId);

    case "screen_employee_compliance":
      // Call Locus pay-per-use OFAC screening endpoint
      return screenCompliance(input.employeeId as string, input.walletAddress as string);

    case "execute_payment":
      const payment = await sendPayment({
        fromWalletId: run.company.locusWalletId,
        toAddress: input.walletAddress as string,
        amountUsdc: input.amountUsdc as number,
        memo: JSON.stringify({ employeeId: input.employeeId, runId: run }),
        idempotencyKey: input.idempotencyKey as string,
        justification: input.justification as string,
      });
      await prisma.payment.update({
        where: { idempotencyKey: input.idempotencyKey as string },
        data: { locusPaymentId: payment.id, status: "SUCCESS", settledAt: new Date() },
      });
      return payment;

    case "flag_employee_for_review":
      await prisma.employee.update({
        where: { id: input.employeeId as string },
        data: { complianceStatus: "FLAGGED" },
      });
      return { flagged: true };

    case "write_audit_entry":
      await prisma.auditLog.create({
        data: { payrollRunId: run.company.locusWalletId, action: input.action as string, details: input.details as object, agentStep: 0 },
      });
      return { logged: true };

    default:
      throw new Error(`Unknown tool: ${name}`);
  }
}
```

```typescript
// Streams real-time agent activity to frontend
export async function GET(req: Request, { params }: { params: { runId: string } }) {
  const encoder = new TextEncoder();
  const stream = new ReadableStream({
    async start(controller) {
      // Poll audit logs and stream updates
      const interval = setInterval(async () => {
        const logs = await prisma.auditLog.findMany({ where: { payrollRunId: params.runId }, orderBy: { createdAt: "asc" } });
        controller.enqueue(encoder.encode(`data: ${JSON.stringify(logs)}\n\n`));
      }, 1000);
      req.signal.addEventListener("abort", () => clearInterval(interval));
    }
  });
  return new Response(stream, { headers: { "Content-Type": "text/event-stream" } });
}
```

```typescript
// Locus calls this on payment settlement
export async function POST(req: Request) {
  const signature = req.headers.get("locus-signature")!;
  const body = await req.text();
  
  // Verify webhook signature
  verifyLocusWebhook(body, signature, process.env.LOCUS_WEBHOOK_SECRET!);
  
  const event = JSON.parse(body);
  if (event.type === "payment.settled") {
    await prisma.payment.update({
      where: { locusPaymentId: event.data.id },
      data: { status: "SUCCESS", settledAt: new Date(event.data.settledAt) }
    });
  }
  return Response.json({ received: true });
}
```

## 🏆 Hackathon Winning Strategy

### What judges will see
1. **Demo trigger**: Click "Run Payroll" — agent starts live, logs appear in real time
2. **Compliance drama**: One employee gets flagged — agent blocks payment, explains why
3. **Real settlement**: 4 employees receive USDC — show Locus transaction IDs, on-chain proof
4. **Audit trail**: Every agent decision logged with justification — full transparency
